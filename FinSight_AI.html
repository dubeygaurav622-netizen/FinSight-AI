Core Capabilities

Everything you need for
Financial Document AI

End-to-end system covering secure authentication, intelligent document storage, AI-powered retrieval, and fine-grained access control.

🔐

JWT Authentication

Secure token-based authentication with refresh tokens, user registration, login endpoints, and stateless session management.

📁

Document Management

Upload, retrieve, and manage financial documents — invoices, reports, contracts — with rich metadata and full CRUD operations.

🧠

RAG Semantic Search

Retrieval-Augmented Generation pipeline that chunks documents, generates embeddings, stores in Qdrant, and returns the most relevant context.

🎯

Smart Reranking

Two-stage retrieval: vector search returns Top 20 candidates, then a reranker narrows to Top 5 most financially relevant results.

🛡️

Role-Based Access Control

Four-tier RBAC system — Admin, Analyst, Auditor, Client — with granular permission enforcement on every endpoint.

⚡

Vector Database Integration

Native integration with Qdrant (or FAISS/Chroma) for lightning-fast similarity search over large financial document corpora.

RAG Architecture

Document Processing Pipeline

Each uploaded document flows through an intelligent processing chain before being available for semantic retrieval.

1

Upload

Document received via POST /documents/upload

2

Text Extraction

Parse PDF/DOCX, extract raw text content

3

Chunking

Split into semantic chunks via LangChain

4

Embeddings

Generate vector embeddings for each chunk

5

Vector Store

Persist vectors in Qdrant with metadata

Retrieval Flow

Search & Reranking Pipeline

Q

User Query

POST /rag/search with query text

E

Embed Query

Convert query to dense vector

20

Vector Search

Retrieve Top 20 from Qdrant

↑

Rerank

Finance-aware reranking model

5

Top Results

Return 5 most relevant chunks

// Sample RAG Search Request
POST /rag/search

{
  "query": "financial risk related to high debt ratio",
  "top_k": 5,
  "rerank": true
}

// Response — returns most relevant financial document chunks
{
  "results": [
    { "chunk_id": "doc_3_chunk_7", "score": 0.94, "text": "..." },
    { "chunk_id": "doc_1_chunk_2", "score": 0.91, "text": "..." }
  ]
}

Access Control

Role-Based Permissions

Four distinct roles with carefully scoped permissions ensure secure, auditable access to financial documents.

Admin

👑 Full Access
  • ✓ Manage all users & roles
  • ✓ Upload & delete documents
  • ✓ Access all RAG endpoints
  • ✓ View audit logs
  • ✓ Configure system settings

Financial Analyst

📊 Analyst
  • ✓ Upload documents
  • ✓ Edit own documents
  • ✓ Perform semantic search
  • ✓ View all documents
  • ✗ Delete documents

Auditor

🔍 Review Only
  • ✓ View all documents
  • ✓ Search & retrieve context
  • ✓ View document metadata
  • ✗ Upload documents
  • ✗ Modify any data

Client

👤 View Only
  • ✓ View company documents
  • ✓ Basic semantic search
  • ✗ Upload documents
  • ✗ Edit or delete
  • ✗ Admin operations
